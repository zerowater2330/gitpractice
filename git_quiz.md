# git 기초 Quiz
1. git의 사전적 정의는 "(중앙) 버전 관리 프로그램"이다.
   - 답 : 분산 버전 관리 프로그램이다.
   - 이유 : 여러 특정 상태의 프로그램을 저장, 공유, 관리하는 프로그램이다.
2. git의 3공간 working directory, staging area, repository 이다.
   - 답 : working directory, staging area, commits
   - 이유 : git의 3가지 공간은 working directory, staging area, repository
3. github에 버전을 저장하기 위해 git commit 명령어를 쓴다.
   - 답 : git commit
   - 이유 : 버전을 저장을 위한 명령어는 git commit이다.
4. git log를 통해 버전 기록들의 목록을 확인할 수 있다.
   - 답 : 버전 기록 목록을 볼 수 있다.
   - 이유 : 버전 기록과 커밋 내역을 확일 할 수 있다.
5. 버전을 기록하는 프로세스 git add -> git commit 이다.
   - 답 : git add -> git status -> git commit 혹은 git commit -m "" -> git log 이다.
   - 이유 : 바로 버전을 업데이트 할 수 있지만 중간에 상태를 확인하면서 커밋 하도록 한다.
6. 변경사항은 6하원칙에 따라 직접 남겨야 한다.
